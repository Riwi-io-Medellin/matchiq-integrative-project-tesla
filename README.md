# 1. General objective
Develop a web platform that automates the pre-selection process for technological
talent, connecting companies with developers through intelligent matching systems
(matching generate in database), automated ranking and technical evaluation,
eliminating manual application, allowing companies to focus only on the best
candidates generated by the system and generates a test with AI.
# 2. Specific objectives
● Implement a secure authentication system with JWT, differentiated roles
(Admin, Company, Candidate), email verification by OTP, password
recovery and guardian implementation with http only.
● Create a job offer management module that allows companies to create, edit
and close offers with categories, skills, English level, modality and salary.
● Develop an intelligent matching engine that, through algorithmic scoring
and AI evaluation, generates a ranking of candidates based on skills,
experience and profile.
● Implement a technical evaluation system (Test) with generation of questions,
sending invitations to candidates, submission of answers and ranking by
score.
● Develop an administration panel with metrics dashboard, user management
(suspension/activation), and catalog CRUD (categories and skills).
● Build complete profiles for candidates with personal data, experience,
seniority level, English level, GitHub, categories and skills selectable from
the catalog.
● Integrate email notifications through n8n + Gmail to inform candidates
about their status (approved, rejected, technical test).
● Deploy the platform with frontend in Netlify and backend in Railway, guaranteeing
availability and scalability
3. Statement of the problem.
In the technology industry, developer recruitment processes face multiple
inefficiencies:
● Overwhelming volume of applications: Companies receive hundreds of
applications per offer, making it humanly impossible to evaluate each profile
fairly and completely.
● Lack of automated matching: Traditional job portals depend on basic filters
(keywords) that do not evaluate the real compatibility between the
candidate's skills and the requirements of the offer.
● Fragmented technical evaluation: Technical tests are managed on separate
platforms, creating friction and loss of context in the process.
● Manual communications process: Notifying each candidate about their status
(approved, rejected, invited to test) consumes significant time of the HR
team.
● Lack of visibility for candidates: Candidates have no way of knowing how
compatible they are with available offers, nor do they receive timely
feedback.
MatchIQ solves these problems by centralizing intelligent matching, technical evaluations
and automated communications in a single platform, reducing pre-selection time.
